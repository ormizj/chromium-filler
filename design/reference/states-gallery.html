Soft / Warm — the parts the mockups didn't cover

The three exploration mockups showed the popup, the two modal views, and one options tab. This page fills the gaps: the modal's other flows, the minimized pill and session strip, the overflow menu, the in-app help disclosure, and the whole setup panel — all in the chosen Soft/Warm language, so a fresh agent has a visual target for every state, not just the happy path.

Correctness note. Status dots carry an icon as well as a colour — filled, low-confidence, unmatched. These are real icons (check / alert / cross), not the text glyphs the exploration mockups used. Two rules survive the change: status is never colour alone (colour-blindness, greyscale need a distinct shape), and the E2E suite asserts on the .cf-dot.none class — so the class stays, only the ::before content becomes a masked SVG.
